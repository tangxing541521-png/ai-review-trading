# trend_score 调试报告 20260622

## 结论

trend_score 没有被写死为 100。最新趋势池共 100 只，分数最小值 86，均值 92.56，最大值 100。

但评分系统存在明显“顶部饱和”：7 只股票得到 100 分，原因是多个子项很容易同时打满，最后又被 `min(100, 子项合计)` 截断。Forward Test 的 Top5 全部 100，不是因为所有股票都 100，而是排序最靠前的股票都触发了封顶。

## 评分公式日志

```text
ma_score = 20 if close > ma5 > ma10 > ma20 else 0
trend_raw = pct_5d * 0.5 + pct_10d * 0.45 + pct_20d * 0.35
trend_score_part = min(25, max(0, trend_raw))
high_raw = 15 * (1 - distance_to_20d_high / 10)
high_score = min(15, max(0, high_raw))
liquidity_raw = amount / 3_000_000_000 * 20
liquidity_score = min(20, max(0, liquidity_raw))
risk_penalty = 4 if pct_5d > 30 else 0
             + 3 if pct_chg > 8 else 0
             + 2 if distance_to_20d_high > 8 else 0
risk_score = max(0, 10 - risk_penalty)
today_score = 10 if 1 <= pct_chg <= 5
            = 6  if 0 < pct_chg < 1
            = 7  if 5 < pct_chg <= 8
            = 3  otherwise
trend_score = round(min(100, ma_score + trend_score_part + high_score + liquidity_score + risk_score + today_score), 2)
```

## 逐项诊断

1. trend_score 是否被写死为 100：否。趋势池分数分布为 86 到 100，只有 Top 7 被封顶到 100。
2. 各子指标是否没有参与计算：否。逐股复算的 calc_score 与文件 file_score 基本一致，说明均线、趋势、新高、流动性、风险、今日涨幅都参与了计算。
3. 是否所有股票都满足满分条件：否。只有 7/100 只股票最终 100，但大量股票在均线、趋势强度、新高距离、流动性等单项上打满。
4. 是否归一化逻辑错误：不是 min/max 归一化错误，而是阈值封顶过早。趋势强度超过 25 后都记 25，成交额超过 30 亿后都记 20，距离 20 日新高为 0 时都记 15。
5. 是否所有风险扣分没有生效：风险扣分逻辑存在并生效，但条件较少且偏温和。当前 72/100 只股票无风险扣分；Top7 的 pct_5d、pct_chg、distance 都没有触发扣分，所以风险项全为 10。

## 关键原因

Top5 全部 100 的直接原因是：

- 均线多头排列普遍满分：Top 股全部 close > ma5 > ma10 > ma20。
- 趋势强度公式很容易超过 25 分上限。
- 处在 20 日新高附近时，新高项容易满分。
- 成交额超过 30 亿就流动性满分，Top 股普遍满足。
- 风险扣分只检查 pct_5d > 30、pct_chg > 8、distance > 8，Top 股多数未触发。
- 最终总分用 min(100, 子项合计) 截断，导致强股之间的差异被压扁。
